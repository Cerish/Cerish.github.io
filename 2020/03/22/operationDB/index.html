    1.2数据库的列类型

    数值

    • tinyint 十分小的数据 1个字节
    • smallint 较小的数据 2个字节
    • mediumint 中等大小的数据 3个数据

    2.2DML语言

    DML语言： 数据操作

    • insert

      • insert into 表名 （字段1，…） values (字段1， …) ==一一对应==
      • insert into 表名 values (字段1， 字段2， ….) ==前面字段不写默认按表的排序，缺一不可==

    • update

      • update 表名 set 字段1 = value [where id = 1] ==不写更改所有数据==

    • delete

      • delete from 表名 [where id = 1] ==不写删除整个表的数据，不破坏表结构==
      • truncate 表名 ==删除整个表的数据，不破坏表结构==
      • truncate 与delete 差别在自增==归置默认==，事务。

    3 事务

    事务原则（事务提交不可逆）

    ==ACID原则==

    • 原子性 要么都成功，要么都失败。

    • 一致性 结果一致性(过程一致性)

    • 隔离性 多个事务互不干扰

      隔离带来的问题

      • 脏读: 一个事务读取到另外一个事务未提交的数据。
      • 不可重复读 : 在一个事务内读取某一行数据，多次读取结果不同。
      • 幻读(虚读) : 是指在一个事务内读到了别的事务插入的数据，导致前后读取不一致。

    • 持久性

      • 事务没提交， 恢复到原有状态
      • 事务提交，持久化到数据库

    7 数据库设计

    三大范式

    • 第一范式(1 NF)
      • 原子性： 数据表的每一列不可再分。
    • 第二范式(2 NF)
      • 前提： 满足第一范式
      • 每张表只描述一件事情
    • 第三范式(3 NF)
      • 前提： 满足第一范式和第二范式
      • 确保数据表中的每一列数据和主键直接相关， 而不能间接相关。
