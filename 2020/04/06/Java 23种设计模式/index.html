    Java 23种设计模式

    1. 设计模式的七大原则（principle）

    • 单一职责原则： 一个类应该只负责一项职责。（例如：mapper 只连接一个表的数据）

    • 接口隔离原则：一个类对另一个类的依赖应该建立在最小的接口上。（ 实现类应该只有需要实现的接口）

    • 依赖倒转原则：中心思想是 面向接口编程。抽象不应该依赖细节， 细节应该依赖抽象。

    • 里氏替换原则： 尽量少的重写父类方法。（采用 依赖、聚合、组合的方式替代继承关系）

    • 迪米特原则：对直接朋友开放， 只与直接朋友通信。

    • 开闭原则： 对扩展开放， 对修改关闭。

    • （合成复用原则）

    2. UML类图

    类之间的关系

    • 依赖 （Dependence）
      • 类中用到了对方
      • 类的成员属性
      • 方法的返回值类型
      • 方法的接收参数类型
      • 方法中使用到
    • 泛化（ generalization 继承）
      • 实际上就是继承关系
    • 实现 （Implementation 接口实现）
      • 实际上就是接口实现
    • 关联 （Association n 对 m）
      • 单向
      • 双向
    • 聚合（ Aggregation setter方法注入）
      • 对象之间的关系可以分离
    • 组合（Composition 属性直接实例化）
      • 对象之间的关系不可分离

    3.3 抽象工厂模式（产品族）

    3.3.1 工厂方法与抽象工厂的区别

    • 工厂方法 ：用来生产同一等级结构中的固定产品。（支持拓展增加产品）

    • 抽象工厂 ：用来生产不同产品族的全部产品。（不支持拓展增加产品；支持增加产品族）

    3.4 原型模式

    有两种深度拷贝的方式

    • Cloneable 接口复制， 默认浅拷贝， 堆内存需要手动在一次调用clone方法
    • 实现 Serializable 接口进行序列化深度拷贝，推荐使用

    3.5 建造者模式

    建造者模式的四个角色

    • Product（产品角色）： 具体的产品对象。
    • Builder（抽象建造者）： 创建一个具体流程的接口/抽象类。
    • ConcreteBuilder（具体建造者）： 实现类， 完成各个具体的流程。
    • Director（指挥者）： 具体指挥产品的构建，隔离用户与创建的对象。

    ​ ==与抽象工厂模式相比，建造者模式返回一个组装好的完整产品，而抽象工厂模式返回一系列相关的产品，这些产品位于不同的产品等级结构，构成了一个产品族 。==

    3.16 迭代器模式

    迭代器模式：提供一种方法顺序访问一个集合对象中的各种元素，而又不暴露该对象的内部表示.

    事先知道数据的存储结构，对外提供一个方法，遍历输出每项元素，内部实现next,hasNext等方法。

    3.19 备忘录模式

    备忘录模式是一种行为模式，在不破坏封闭的前提下，将对象当前的内部状态保存在对象之外，之后可以再次恢复到此状态。典型的例子就是游戏存档和读档的这个行为。

    3.20 解释器模式

    解释器模式（Interpreter），给定一个语言，定义它的文法的一种表示，并定义一个解释器，这个解释器使用该表示来解释语言中的句子。例如计算表达式，给定 a+b-c， 按照解释器的语言来解释执行。
