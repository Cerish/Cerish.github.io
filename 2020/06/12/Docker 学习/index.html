10. CI（持续集成） Jenkins+Gitlab+Docker

配置ssh服务器

  • 首页 -> 点击系统管理 -> 管理插件 ->可选插件 -> 过滤：ssh -> 选择Publish Over SSH插件，点击直接安装

  • jenkins首页，系统管理 -> 系统设置 -> 下拉，找到Publish over SSH 点击新增

    • Name为服务器名称
    • Hostname填写服务器ip
    • userName填写用户名
    • RemoteDirectory指工作目录,比如填写/tmp 发送过去的所有文件默认在/tmp下

创建任务

输入任务名称 –> 选择构建一个maven项目 (如果没有这个选项， 那就需要下载插件==Maven Integration plugin==)

  • 在general中选择丢弃旧的构建以节省磁盘资源

  • 源码管理配置 git账户名和密码

  • 构建触发器

    • Build whenever a SNAPSHOT dependency is built
    • GitHub hook trigger for GITScm polling

  • 构建环境

  • Add timestamps to the Console Output

  • Build

    • pom.xml
    • clean install package -Dmaven.test.skip=true -Dmaven.wagon.http.ssl.insecure=true -Dmaven.wagon.http.ssl.allowall=true -Dmaven.wagon.http.ssl.ignore.validity.dates=true

  • Post Steps

  • Run only if build succeeds

  • 构建后操作
